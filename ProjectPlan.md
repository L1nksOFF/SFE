# План управління проектом: "Student File Exchange"

## 1. Мета:

**Продукт:** Десктопний додаток "Student File Exchange".

**Мета:** Розробити зручний та надійний десктопний додаток на JavaFX для студентів. Продукт має вирішити проблему незручного обміну навчальними матеріалами, інтегрувавши додаток з Google Drive API для безкоштовного та надійного хмарного зберігання файлів (згідно з ТЗ).

## 3. Етапи: основні фази розробки

Проект розділений на 3 основні ітерації:

* **Ітерація 1 (MVP):** Налаштування архітектури та середовища; Інтеграція ключових технологій: JavaFX, SQLite та Google Drive API; Реалізація базового функціоналу: автентифікація користувача та завантаження файлу.
* **Ітерація 2 (Розширення):** Реалізація логіки скачування файлів; Створення та відображення ієрархічної структури категорій (Спеціальність > Курс > Предмет); Реалізація повноцінного перегляду файлів за категоріями.
* **Ітерація 3 (Полірування):** Додавання вторинного функціоналу (генерація посилань, рейтинг файлів); Забезпечення кросплатформовості та адаптивності інтерфейсу.

## 4. Строки: орієнтовний графік виконання

Загальна орієнтовна тривалість розробки проекту Student File Exchange складає **6 тижнів**. Робота структурована у три 2-тижневі ітерації, кожна з яких фокусується на конкретних пріоритетах наведених у Product Backlog.

* **Тижні 1–2 (Ітерація 1): Налаштування та MVP**
    * **Мета:** Створення основи та базового функціоналу.

* **Тижні 3–4 (Ітерація 2): Розширення основного функціоналу**
    * **Мета:** Додавання можливостей для взаємодії з контентом.

* **Тижні 5–6 (Ітерація 3): Полірування та нефункціональні вимоги**
    * **Мета:** Завершення вторинного функціоналу, розробка повноцінного інтерфейсу та забезпечення якості.

Цей 6-тижневий план дозволяє послідовно рухатися від MVP до повноцінного, відполірованого продукту.

    * *Стратегія:* Фокус на одній ОС для розробки MVP; Використання фреймворку адаптованого під кросплатформенність.

## 6. Критерії якості: як оцінюватиметься готовність

Готовність кінцевого продукту "Student File Exchange" визначається відповідністю наступним ключовим критеріям:

1.  **Функціональна повнота:**
    * Усі завдання, визначені у Product Backlog, реалізовані, протестовані та працюють коректно.
    * Основний функціонал додатку (завантаження, категоризація, скачування, обмін) функціонує без збоїв.

2.  **Відповідність нефункціональним вимогам:**
    * **Надійність:** Додаток стабільно працює, а інтеграція з Google Drive API забезпечує надійне зберігання файлів.
    * **Кросплатформовість:** Продукт протестований і коректно функціонує на цільових ОС (Windows та Linux).
    * **Користувацький досвід (UI/UX):** Інтерфейс є інтуїтивно зрозумілим, відповідає вимогам адаптивності та не потребує тривалого навчання для студентів.

3.  **Якість коду та архітектури:**
    * Вихідний код є чистим, структурованим та добре організованим.
    * Архітектура додатку дозволяє легке виправлення помилок та поступове майбутнє розширення функціоналу.

4.  **Готовність до використання:**
    * Продукт є самодостатнім десктопним додатком, який можна запустити та використовувати безпосередньо на комп'ютерах користувачів.
